Решение

Интерфейс пользователя построен на основе терминала без использования графического интерфейса.

Структура меню сделана подобной аналогичным системам, но построенным на графическим интерфейсе.

По предполагаемой функциональности должна была перекрыть весь комплекс задач, решаемых подобными системами с интерфейсом к системам видеонаблюдения.

Учитывая опыт использования аналогичных систем акцент был сделан на простоте ввода большого количества данных при первоначальном запуске в работу. На практике необходимо ежемоментно описать работу порядка нескольких тысяч источников информации и связать их логически с тысячами исполнительных устройств, регистраторов.

Основной проблемой при запуске подобных систем является время необходимое для прописывания всех взаимосвязей и их корректировки по результатам тестирования. Из практики это занимает около мясца работы двух специалистов не менее 12 часов в день с использованием графического интерфейса.

Обычно информация вводится с рукописных записей или напрямую из проектной документации, из чертежей (рисунков). При этом одной рукой специалист двигает линейку или что нибудь аналогичное по рукописным записям а второй вводит информацию.

Известно, что самым быстрым является клавиатурный ввод без использования мыши. Учитывая специфику ввода данных в курсовой для ввода данных используется цифровое поле клавиатуры, а там где было недостаточно кнопок цифрового поля использовались кнопки основного поля не чувствительные к регистру и раскладке клавиатуры.

При разработке меню программы была предусмотрена возможность наряду с движением по дереву, также и горизонталные переходы. Данная возможность до конца не реализована ( не прописаны переходы ), поскольку это больше задача проектирования эргономического интерфейса нежели проблема программирования.

Реализация

Учитывая, что данная задача достаточно универсальна, код был выстроен по типу шаблона, где с минимальными затратами можно заменить меню и функциональность на другую область применения. Для этого было сделано:

  • Основное меню было сформировано как массив (private static final MainMenu[]) элементов record MainMenu(int jItem, int jNext, int level, String uTitle) Предпочтение было отдано массиву, а не списку, поскольку он на протяжении жизни программы неизменяем и загружается один раз MainMenu.init() при старте интерфейса MainMenu.consol();

    Параметр int jNext зарезервирован для реализации горизонтального перехода в меню, int level используется при определении доступности элемента меню в зависимости от прав пользователя, String uTitle-текст меню, int jItem-номен пункта меню

    В систему заложено ограничение, где пункты меню могут быть от 1 до 8. 9-это переход на верхний уровень меню, а 0-из любой точки переход не самый верхний уровень.

    Такой подход позволяет все изменения структуры меню, наименований делать только в одном месте кода программы - MainMenu.init(){}

  • Все действия вызываемые из главного меню формируются в actions(Scanner con, int x) через switch (jSelect), в котором требуемым пунктам меню сопоставлен вызов соответствующего метода.

  • В программе создан единый диалог (Dialog.uConsol(con, SensoRoom.list),где con-передача ссылки на объект консоль, а SensoRoom.list, для примера, - ссылка на список объектов, который подлежит корректировке.) для работы со всеми возможными списками. Для этого создан интерфейс public interface Item с набором всех необходимых методов для реализации единообразного подхода ко всем спискам.

    Данный диалог позволяет добавлять элементы в список, редактировать и удалять. Функционально он может делать групповые действия для заданного диапазона. Применеие групповых действий должно значительно повысить производительность пользователя при вводе подобных элементов. Групповые операции выполняются с использованием трафарета.

    Трафарет позволяет задать какие поля/параметры каждого элемента будут задаваться или редактироваться, а также установить для них требуемое значение.

  • Для реализации изменения параметров объекта либо трафарета создан класс class XFields. Данный класс обеспечивает единообразный подход к работе с различными объектами. Для этого для каждого типа объекта, что равносильно классу системы, в описании объекта создается массив, например static public XFields[] uField = new XFields[3];, описывающий все параметры объекта подлежащие редактированию.

    В классе XFields реализован поддержка следующих типов параметров объекта: enum Typ{INT,STRING,ENUM,LIST}, а именно поддержка целочисленных значение, строковых, возможности выбора из ограниченного списка, создания ссылки на элемент из другого списка. Приведенный набор типов параметров элемента позволяет создать полноценную базу данных описания объектов.

    Массив XFields[] инициализируется в момент инициализации первого элемента списка.

    В процессе разработки программы подходы к работе с элементами списка менялись и на реализации объекта public class SensoRoom implements Item была сделана реализация, позволяющая преобразовать class к record для уменьшения объема кода. А именно было сделано разделение на статические и динамические данные объекта. В частности в классе class SensoRoom динамические данные были собраны в класс QSensor status; //состояние датчика

  • Сохранение данных в файл и восстановление из него реализовано посредством класса class UtilFile и связанного мс ним перечисления enum GrRecords. Данный подход взят из моего предыдущего проекта методом обрезания функциональности, и поэтому там сохранилась некоторая избыточность.

    Система сохраняет данные в текстовый файл из всех списков. Перечень всех списков определен в enum GrRecords.

    Восстановление данных посходит чтением фала до метки ENDFL. При этом данные списков могут располагаться в произвольном порядке. Система сама определяет к какому списку относится конкретная строка текста.

Из всего необходимого объема данных для работы пожарно-охранной сигнализации реализовано только несколько списков. Дальнейшая реализация функционала не требует большой разработки кода, а максимально использует имеющиеся наработки. Дальше будет копирование и изменение отдельных значений.
